# Video Clip Cutter

長編動画（16:9）から短編動画（9:16）を一括生成するスキル。
1本の出力動画に対して複数パートの結合が可能（マルチパート対応）。

## 再実行モード（最優先）

「再度書き出して」「もう一度書き出して」「〇〇座だけ書き出して」など、
**CSVや動画について新しい情報を何も提示していない場合は、既存の `clips/clips.csv` と `clips/input.mp4` をそのまま使って即実行する。**
STEPを踏み直す必要はない。確認も不要。すぐ `batch_clip.py` を実行すること。

## /clip コマンド（インタラクティブモード）

ユーザーが `/clip` と入力したとき、または新しい動画・要約テキストを提示したときに使う。
**各ステップで必要な情報を1つずつ聞き、前のステップが完了してから次に進むこと。**

```
STEP 1: 入力動画の確認
  → 「YouTube動画のリンクを貼ってください。ローカルファイルの場合は clips/input.mp4 に置いてあればOKです。」

STEP 2: 要約テキストの取得
  → 「動画の要約テキストを貼ってください。各星座（またはセクション）のタイムスタンプ付きで。」

STEP 3: 日付の確認
  → 「何日〜何日の運勢ですか？（例: 4/6〜4/12）」

STEP 4: CSV自動生成 → 確認
  → 要約テキストから自動分割アルゴリズムでCSVを生成
  → 「このCSVで進めていいですか？修正があれば教えてください。」
  → CSVの内容を表示してユーザーの確認を待つ

STEP 5: 実行
  → ユーザーがOKしたら clips/clips.csv に保存
  → YouTubeリンクがあれば yt-dlp でダウンロード
  → batch_clip.py を実行
  → 完了後、出力ファイル一覧を表示
```

**注意:**
- STEP 4 でユーザーの確認を得るまで実行に進まないこと
- 修正依頼があればCSVを修正して再度確認を取る
- フォントパスは前回使ったものを覚えていればそれを使う。初回のみ確認する

## 出力仕様（デフォルト）

| 項目 | 値 |
|------|-----|
| 出力解像度 | 1080x1920（9:16） |
| 映像配置 | 中央クロップ（元動画の中央を正方形に切り出し → 中段に配置） |
| 上下余白 | **ぼかし背景（デフォルト）** / 黒 / 白 の3択（`--bg blur/black/white`） |
| テキスト | 太字、80px、色 #ffffff（白）、黒枠 1px |
| パート長 | CSVの start〜end で可変指定。1出力あたり合計約52秒 |
| オープニング | なし（動画は最初から始まる） |
| エンドカード | 末尾に5秒間表示 |
| コーデック | H.264 (libx264) / AAC |

## 作業ディレクトリ（固定）

```
Documents/claude/2026/oshi-uranai/clips/
├── input.mp4       ← 毎週ここに長尺動画を置く
├── clips.csv       ← 毎週ここに切り出しリストを置く
├── endcard.png     ← エンドカード画像（1080x1920、固定）
└── output/         ← 短編動画がここに書き出される
```

特にパス指定がなければ、上記を自動的に参照すること。
`output/` フォルダが存在しなければ自動作成する。

## 入力動画の取得

### パターンA: ローカルファイル（デフォルト）
`clips/input.mp4` に直接配置。

### パターンB: YouTubeリンク
ユーザーがYouTubeのURLを指定した場合、`yt-dlp` で自動ダウンロードする。

```bash
# インストール確認（なければ pip install yt-dlp）
yt-dlp --version

# ダウンロード（mp4、最高画質のH.264）
yt-dlp -f "bestvideo[ext=mp4][vcodec^=avc]+bestaudio[ext=m4a]/best[ext=mp4]" \
  -o "clips/input.mp4" \
  --merge-output-format mp4 \
  "{youtube_url}"
```

ダウンロード完了後、通常の切り出し処理に進む。

## CSVフォーマット

### 列定義

```csv
output,start,end,top_text,bottom_text
```

| 列 | 説明 |
|----|------|
| output | 出力ファイル名。同じ名前の行はすべて順番に結合される |
| start | パートの開始時間（HH:MM:SS） |
| end | パートの終了時間（HH:MM:SS） |
| top_text | 上部キャプション。`|` で改行（パイプ文字） |
| bottom_text | 下部キャプション。`|` で改行（パイプ文字） |

### テキストルール
- 改行は `|`（パイプ文字）で指定。**`\n` は使わない**
- 1行あたり全角12文字以内を推奨（超えると左右がみきれる）

### 例1: 12星座用（各星座マルチパート）

```csv

clip01_taurus.mp4,00:00:42,00:01:20,4/6〜4/12|牡牛座の運勢,価値観の刷新と|調和がテーマ
clip01_taurus.mp4,00:02:10,00:02:50,4/6〜4/12|牡牛座の運勢,直感に従って動く
clip02_gemini.mp4,00:03:58,00:04:40,4/6〜4/12|双子座の運勢,内面を見つめる|深い言語化の時
clip02_gemini.mp4,00:05:00,00:05:30,4/6〜4/12|双子座の運勢,言葉が降りてくるのを|待つのが吉
```

### 例2: 個別星座の詳細リーディング（1星座マルチパート）

```csv
output,start,end,top_text,bottom_text
clip_aries.mp4,00:00:30,00:00:56,4/6〜4/12|牡羊座の運勢,前半は直感を|信じる時期

clip_aries.mp4,00:03:45,00:03:50,4/6〜4/12|牡羊座の運勢,タロットカードの|メッセージ

```

### 例3: 1パートのみ（シンプル）

```csv
output,start,end,top_text,bottom_text
clip01_taurus.mp4,00:00:42,00:01:42,4/6〜4/12|牡牛座の運勢,価値観の刷新と|調和がテーマ
```

### グルーピングルール
- CSVを上から読み、**同じ `output` 名が連続する行をひとまとめ**にする
- `output` 名が変わったら別の動画として処理する
- 各グループの全パートを結合した後、末尾にエンドカードを付加する

## YouTube要約からCSVを自動生成するルール

ユーザーがYouTube動画の要約テキストを貼った場合、以下のルールで**自動的に**CSVを生成する。
**ユーザーに区間の指定を求めてはいけない。AIが自動計算すること。**

**⚠️ 最重要: 1星座＝1行（1パート）にしてはいけない。必ず内容を複数パートに分割すること。**

### 出力動画の目標尺

**1本あたり合計約60秒（オープニング3秒 + コンテンツ約52秒 + エンドカード5秒）**

### 自動分割アルゴリズム

```
入力: 要約テキスト（各星座にタイムスタンプ付き）
出力: マルチパートCSV（1出力あたり合計約52秒のコンテンツ）

STEP 1: 各星座のタイムスタンプと次の星座のタイムスタンプから「持ち時間」を算出
  - 例: 牡牛座(0:42)、双子座(3:58) → 牡牛座の持ち時間 = 3:58 - 0:42 = 3分16秒
  - 最後の星座は、持ち時間を直前の星座と同じ長さと推定する

STEP 2: 要約テキストの文（句点「。」区切り）の数をカウント → パート数とする
  - 文が1つしかない場合 → 2パートに分割
  - 文が2つ → 2パート
  - 文が3つ以上 → 最大3パート（文をグルーピング）

STEP 3: コンテンツ予算52秒をパート数で均等割り → 各パートの長さを算出
  - 2パート → 各約26秒
  - 3パート → 各約17秒
  - 各パートの最低長さは10秒とする

STEP 4: 各パートの start/end を持ち時間の中から配置する
  - パート1の start = 星座の開始タイムスタンプ + 3秒（導入を少し飛ばす）
  - パート1の end = パート1の start + パート長
  - パート2以降は、持ち時間を均等に区切った位置から開始する
    （例: 持ち時間3分、2パートなら、パート2は持ち時間の中間地点から開始）
  - パート間に数秒の間を空けることで、場面転換感を出す

STEP 5: 各パートの bottom_text を、対応する文の要約から生成
  - 全角12文字×2行以内に収める
  - 内容が異なるキャプションにする（全パート同じにしない）
```

### 具体例

**入力（要約テキスト）：**
```
牡牛座 (0:42)： 価値観の刷新と調和がテーマ。直感に従って動くことで、より豊かな暮らしへ。
双子座 (3:58)： 内面を見つめる「深い言語化」の時。前半は静かに過ごし、言葉が降りてくるのを待つのが吉。
```

**自動計算の過程：**
```
牡牛座: 開始=0:42, 次の星座=3:58, 持ち時間=3:16
  文の数=2 → 2パート、各パート27秒
  パート1: 0:45 〜 1:12 (開始+3秒から27秒間、テーマ紹介)
  パート2: 2:20 〜 2:47 (持ち時間の中間地点から27秒間、具体的アドバイス)

双子座: 開始=3:58, 次の星座=7:52, 持ち時間=3:54
  文の数=2 → 2パート、各パート27秒
  パート1: 4:01 〜 4:28 (開始+3秒から27秒間、テーマ紹介)
  パート2: 5:55 〜 6:22 (持ち時間の中間地点から27秒間、具体的アドバイス)
```

**出力（CSV）：**
```csv
output,start,end,top_text,bottom_text
clip01_taurus.mp4,00:00:45,00:01:12,4/6〜4/12|牡牛座の運勢,価値観の刷新と|調和がテーマ
clip01_taurus.mp4,00:02:20,00:02:47,4/6〜4/12|牡牛座の運勢,直感に従って動き|豊かな暮らしへ
clip02_gemini.mp4,00:04:01,00:04:28,4/6〜4/12|双子座の運勢,内面を見つめる|深い言語化の時
clip02_gemini.mp4,00:05:55,00:06:22,4/6〜4/12|双子座の運勢,静かに過ごし|言葉を待つのが吉
```

**⚠️ 絶対に守ること：**
- 区間の指定をユーザーに聞き返してはいけない。上記アルゴリズムで自動計算する
- 1星座1パートのCSVは絶対に生成しない。最低でも2パートに分割する
- 各パートの bottom_text はそれぞれ異なる内容にする

## 処理手順

**⚠️⚠️⚠️ 最重要ルール ⚠️⚠️⚠️**
**自分でffmpegスクリプトを書くな。同梱の `scripts/batch_clip.py` をそのまま実行すること。**
**このスクリプトは以下をすべて正しく処理する：**
- **テキストの `|` split → 個別drawtext生成（改行表示）**
- **テキストの天地センター配置**
- **オープニング3秒 + パート群 + エンドカード5秒の結合**
- **音声・映像の同期（再エンコード結合）**
- **パート間のフェードイン・フェードアウト（背景色に合わせて自動切替）**
- **オープニングなし（動画は最初から始まる）。`--opening` 指定時のみ冒頭に追加**
- **`--subtitle` 指定時はWhisperで自動文字起こし → タイミング付き字幕を動画中央に焼き込む**

### 1. 環境確認

```bash
ffmpeg -version
python3 --version
```

### 2. 日本語フォント確認

OS別にフォントパスを確認し、`--font` 引数に渡す：

- **Mac**: `/System/Library/Fonts/ヒラギノ角ゴシック W6.ttc`
- **Linux**: `fc-list :lang=ja` で太字フォントを検索
- **Windows**: `C:/Windows/Fonts/meiryo.ttc` または `C:/Windows/Fonts/YuGothB.ttc`

### 3. スクリプト実行

```bash
# 字幕なし（デフォルト）
python3 scripts/batch_clip.py \
  --input clips/input.mp4 \
  --csv clips/clips.csv \
  --endcard clips/endcard.png \
  --outdir clips/output \
  --font "フォントパス" \
  --bg blur

# 字幕あり（Whisper自動文字起こし）
python3 scripts/batch_clip.py \
  --input clips/input.mp4 \
  --csv clips/clips.csv \
  --endcard clips/endcard.png \
  --outdir clips/output \
  --font "フォントパス" \
  --bg blur \
  --subtitle \
  --whisper-model small
```

- `--subtitle`: 字幕を自動生成して動画中央に焼き込む（要 `pip install faster-whisper`）
- `--whisper-model`: `tiny`（高速）/ `small`（デフォルト）/ `medium` / `large-v3`（高精度）
- `--opening` は省略可能。指定した場合のみ冒頭に挿入される（デフォルト: なし）
- `--bg` には `blur`（ぼかし背景・デフォルト）/ `black`（黒）/ `white`（白）を指定できる
これだけで全処理が完了する。自分でffmpegコマンドを組み立てる必要はない。

### スクリプトが行う処理の流れ

```
1. CSVを読み込み、output 名でグルーピング
2. opening.png → 3秒の無音動画に変換
3. endcard.png → 5秒の無音動画に変換
4. output グループごとにループ：
   a. 各パートを切り出し + クロップ + パッド + テキスト + フェード → 中間ファイル
   b. concat リスト作成（opening + 全パート + endcard）
   c. 再エンコード結合で最終出力
   d. 中間ファイル削除
5. 出力確認（ファイルサイズ + ストリーム情報表示）
```

## カスタマイズ対応

`scripts/batch_clip.py` の先頭にある定数を変更する：

```python
FONTSIZE = 80                 # テキストサイズ
FONTCOLOR = "0xffffff"        # テキスト色（白）
FONTBORDERCOLOR = "0x000000"  # テキスト枠色（黒）
FONTBORDERW = 1               # テキスト枠の太さ（px）
LINE_SPACING = 104      # 行間
FADE_DURATION = 0.5     # フェード秒数
OPENING_DURATION = 3    # オープニング秒数
ENDCARD_DURATION = 5    # エンドカード秒数
BG_MODE = "blur"        # "white" / "black" / "blur"
BLUR_SIGMA = 30         # ぼかし強度（blurモード時）
```

その他：
- **背景の指定**: `--bg blur`（ぼかし） / `--bg black`（黒） / `--bg white`（白）
- **出力品質の変更**: `-crf` の値を変更（18=高品質、28=低品質）
