## 実行手順

### 1. 天体データをスクリプトで自動計算する

同梱の `fetch_astro.py`（Skyfield + NASA JPL天文暦によるローカル計算。APIキー・通信不要）を実行して、対象週の天体データを取得する：

```bash
# 初回のみ（Skyfield未導入の場合）
pip install skyfield

# 来週分（日曜〜月曜に翌週分を作るのが通常運用）
python .claude/skills/oshi-uranai-weekly/scripts/fetch_astro.py --next

# 今週分 / 特定週
python .claude/skills/oshi-uranai-weekly/scripts/fetch_astro.py
python .claude/skills/oshi-uranai-weekly/scripts/fetch_astro.py --week 2026-W30
```

出力される内容（すべてJST・トロピカル方式）：
- 全惑星のサイン位置（週初め→週末）＋逆行フラグ
- 月のボイドタイム〜星座入り時刻
- 月相（新月・上弦・満月・下弦）
- 週内の逆行開始・終了
- 主要5アスペクト（オーブ6度以内、持続／週前半のみ／週後半に形成のタグ付き）

**⚠️ どの週を対象にするかを間違えないこと。** ユーザーが日付を指定したらその週に合わせる（`--week` で明示指定が確実）。

### 1-2. 計算結果をユーザーに確認する（1回だけ聞く）

スクリプトの出力サマリーを提示し、**「このデータで進めていいですか？手元のデータと違う点があれば教えてください」と1回だけ確認する。** ユーザーが手動データを提供した場合はそちらを優先して上書きする。確定データはファイル冒頭のメタ情報ブロックに記載する。

**⚠️ astro.com等のWebFetch読み取りは引き続き禁止。** ホロスコープ画像の星座記号を誤読するため信用できない。データ取得は `fetch_astro.py` かユーザー提供データの2択。

### 2. アスペクトをピックアップする

**基本は Step 1 のスクリプト出力のアスペクト一覧を使う。**「持続」タグは週を通じたテーマ、「週前半のみ」「週後半に形成」は前半・後半の色づけに使う（従来の画像読み取りと同じ扱い）。

ユーザーがホロスコープ画像を提供した場合は、補助情報として以下の手順で読み取る：

**使用するアスペクトは以下の5種類のみ（それ以外は無視する）：**
1. コンジャンクション（合・0°）
2. オポジション（衝・180°）
3. スクエア（四分・90°）
4. セクスタイル（六分・60°）
5. トライン（三分・120°）

パラレル・コントラパラレル・セミセクスタイル・クインクス等のマイナーアスペクトは使用しない。

**優先度：** オーブ（誤差角度）が小さいほど優先度が高い。同じアスペクト種でオーブが同程度の場合は、上記の種類順（コンジャンクション＞オポジション＞スクエア＞セクスタイル＞トライン）で優先する。

- 週初め・週末のホロスコープ画像から、主要5アスペクトのうちオーブが小さいものを優先してピックアップする
- 両日で共通しているアスペクト＝週を通じて持続するテーマとして扱う
- 週初めにあって週末にないアスペクト（または逆）＝週の前半・後半で変化するテーマとして扱う
- ピックアップしたアスペクトはメタ情報ブロックの「主要アスペクト」に記載する
- コンテンツ生成時、持続するアスペクトは全体運・各テーマのベースに、変化するアスペクトは「週前半／週後半」の色づけに活用する

### 3. references/aspect_meanings.md を読み込む

### 3-5. 前週のコンテンツを確認する

`oshi-uranai/` フォルダにある直近のmdファイルを読み込み、前週の**タイトル・全体テーマ・締めの問いかけ**を把握する。

今週のコンテンツは前週からの文脈が連続するよう、以下のルールを守ること：
- **タイトルのトーン・言葉が前週と重ならないようにする**（「始まる」「動き出す」「覚悟」などが毎週続くと陳腐化する）
- 前週テーマの「その先」として今週を位置づける（例：「手放した→整った→動き出す→言葉になる→形になる」など季節の文脈で繋ぐ）
- 前週の締めの問いかけに対する「答えが出る」「問いがより深まる」流れを意識するとなお良い

### 4. 以下のコンテンツを**一度に全て**生成する

**【全セクション共通：語り口・トーンのガイドライン】**

以下のスタイルを全セクションで統一すること。

- **短い行で改行**：1〜2文ごとに改行し、読むリズムを作る
- **語尾はカジュアルに**：「〜しそうです」「〜かもしれません」「〜とよさそうです」「〜じゃないですか」「〜なんですよね」「〜ですよね」
- **断定より観察・提案**：「〜です」より「〜かもしれない」「〜してみて」「〜な流れ」
- **占星術用語は平易な言葉に言い換える**：「三重合」→「水星・火星・土星の重なり」、「スクエア」→「ちょっとした緊張感」、「在室」→「入っている」
- **学術的・教科書的な表現を避ける**：「形成します」「届いています」「在室」→「できてます」「影響してます」「入ってます」
- **短い並列で感情を重ねる**：「表現したい / 動きたい / 形にしたい」のように改行で並べる
- **転換は柔らかく**：「ただ、〜」「でも、〜」「ここは〜」で自然に流れを変える
- **締めは余韻を残す**：「静かだけれど、〜」「ちゃんと変化が進んでいるとき」など、押し付けない形で終わる

**NG例（古い・学術的）：** 「言葉・行動・覚悟が完全に揃う瞬間です。」「スクエアを形成する天体が〜」「当該ハウスのテーマ」
**OK例（現代的・カジュアル）：** 「考えてること、動きたい気持ち、腹をくくった想い。その3つが少しずつひとつになる感じです。」「ちょっと引っ張り合う角度になってて〜」

**【文末ルール（必須）】**

- すべての文は言い切りで終えること。許可する文末は次の3つのみ：
  1. 終止形（〜る。／〜た。／〜だ。／〜ない。）
  2. です・ます形
  3. 体言止め（名詞で終わる）
- 接続助詞で文を終えるのは禁止：
  「〜て。」「〜で。」「〜けど。」「〜が。」「〜ので。」「〜から。」「〜し。」「〜たり。」
- 余韻を出したいときは、て形のまま切らず、終止形か体言止めに直す。
  - NG: 出した分の力が、そのまま速度になって返ってきて。
  - OK: 出した分の力が、そのまま速度になって返ってくる。
  - OK: 出した分の力が、そのまま速度になって返ってくる爽快さ。
- 名詞＋助詞で終わる言いさし（「〜の中で。」「〜とともに。」「〜ところに。」「〜の上で。」等）は、**改行・段落の直前では禁止**。直後に同じ文脈の文が改行なしで続く場合のみ使用可。
  - 理由：改行を挟んで別の文脈が始まると、言いさしの意味が宙に浮いて通じなくなるため。
  - 段落・ブロックの最後の文は必ず「言い切り」か「助詞で終わらない体言止め」にする。
- 出力前に全文の文末を確認し、違反があれば修正してから出力すること。

### B節の生成手順（必ずこの順番で処理すること）

**STEP 1：月のハウス別在室スケジュールを計算する**
- 今週、月が通過するサインを時系列で並べる
- 各サインが、その星座にとって何ハウスかを特定する（ホールサイン・ハウス使用）
  - ハウステーマ：1H=自己・2H=価値・3H=対話・4H=家族/土台・5H=創造/喜び・6H=日常/健康・7H=対人・8H=変容・9H=哲学/学び・10H=仕事/社会・11H=友人/目標・12H=内省/見えない領域
- 月の在室期間（何日〜何日、または前半・後半）を明確にする

**STEP 2：週を「ひとつの物語」として組み立てる**

各天体イベント（月のハウス移動・ボイド・フェーズ・スロー天体の関与）を、週という物語の「章」として捉えて組み立てる。読み終えたとき、今週の全体像がひとつの流れとして頭に残るのがゴール。

- **各章には役割を与える**：「整える章」「動く章」「確かめる章」「休む章」など、何のための時間かを明示する
- **章と章をつなぐ**：「〜だから次の章で○○が起きる」「前半で整えたものが後半で○○になる」という因果・伏線の流れを作る。列挙にしない
- **クライマックスを決める**：週の中でいちばん動きやすい・感情が高まるタイミングを特定し、そこへ向けて物語が流れるよう組み立てる（例：上弦の月・金星入座など）
- **ボイドは「場面転換」として使う**：長いボイドは「幕間・休憩」。短いボイドは省略してよい（1時間未満は書かない）
- スロー天体が同じハウスにいる・アスペクトで関与している場合は、その章の「背景テーマ」として加味する

**STEP 3：出力する（フォーマット）**

```
星座名♌
今週の流れ：
  〜XX日（曜）：○○エリア — 短く1文
  XX日〜：△△エリアへ — 短く1文
  ※最大3行まで
気をつけること：（1文）
一言メッセージ

週全体の流れ（YouTube台本）：
  （2段落・150字程度の話し言葉。以下の3ステップ構成で書くこと）
  1. 【冒頭：週全体のテーマを1文で置く】「今週は〇〇がすべての出発点になります」のように、先にテーマを宣言する
  2. 【中盤：各イベントを流れの一部として語る】天体イベントを時系列に並べるのではなく、「その流れが〜」「だからこそ〜」「ようやく〜」などの接続で、前のイベントが次を生み出すように書く
  3. 【締め：週全体を回収する】クライマックス（上弦・金星入座など）で冒頭テーマが「開花・解放・着地」する形で締める
  - この台本だけで内容が完結すること（他セクション参照不要）
  - 天体・アスペクトは略さず書く（「三重合」ではなく「水星・火星・土星の三重合」）
  - ハウスは「3ハウス（対話・発信）」のように番号＋テーマを併記
```

### ファイル保存後の処理

なし（天体データは毎回 `fetch_astro.py` で全量計算するため、前週データの引き継ぎ保存は不要）。
